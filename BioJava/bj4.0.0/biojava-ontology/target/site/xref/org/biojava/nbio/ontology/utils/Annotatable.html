27  /**
28   * <p>Indicates that an object has an associated annotation.</p>
29   *
30   * <p>Many BioJava objects will have associated unstructured
31   * data. This should be stored in an Annotation instance. However, the
32   * BioJava object itself will probably not want to extend the
33   * Annotation interface directly, but rather delegate off that
34   * functionality to an Annotation property. The Annotatable interface
35   * indicates that there is an Annotation property. When implementing
36   * Annotatable, you should always create a protected or private field
37   * containing an instance of ChangeForwarder, and register it as a
38   * ChangeListener with the associated Annotation delegate
39   * instance.</p>
40   *
41   * <pre>
42   * public class Foo extends AbstractChangeable implements Annotatable {
43   *   private Annotation ann; // the associated annotation delegate
44   *   protected ChangeForwarder annFor; // the event forwarder
45   *
46   *   public Foo() {
47   *     // make the ann delegate
48   *     ann = new SimpleAnnotation();
49   *     // construct the forwarder so that it emits Annotatable.ANNOTATION ChangeEvents
50   *     // for the Annotation.PROPERTY events it will listen for
51   *     annFor = new ChangeForwarder.Retyper(this, getChangesupport( Annotatable.ANNOTATION ), 
52   *                                          Annotatable.ANNOTATION );
53   *     // connect the forwarder so it listens for Annotation.PROPERTY events
54   *     ann.addChangeListener( annFor, Annotation.PROPERTY ); 
55   *   }
56   *
57   *   public Annotation getAnnotation() {
58   *     return ann;
59   *   }
60   * }
61   * </pre>
62   * Check if BioJava classes and interfaces extend Annotatable. This
63   * will tell  you if you should look for associated annotation.
64   *
65   *  If an object implements Annotatable, it may well propagate
66   * ChangeEvent notifications from the associated Annotation. You may
67   * need to track these to maintain the state of your applications.
68   *
69   * Be careful to hook up the appropriate event forwarders.
70   *
71   * The getAnnotation() method can be implemented lazily
72   * (instantiate the Annotation instance and event forwarders when the first
73   * request comes in). It can also be implemented by returning throw-away
74   * immutable Annotation instances that are built from scratch each time.
75   * @author  Matthew Pocock
76   * @author <a href="mailto:kdj@sanger.ac.uk">Keith James</a> (docs).
77   * @author  Kalle Nï¿½slund (docs)
78   * @see org.biojavax.RichAnnotatable
79   * @since 1.0
80   */
81  public interface Annotatable  {
82   
83  
84    /**
85     * Should return the associated annotation object.
86     *
87     * @return an Annotation object, never null
88     */
89    Annotation getAnnotation();

91  
92  }
